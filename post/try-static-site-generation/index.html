Try Static Site Generation with Hugo

If you’re into coding or general web development you will have probably thought about creating your own site from scratch at one point or another. If you’ve spent hours trying to decide which CMS system to use, hosting provider to trust, and plugins to use, take a step back for a minute. What if you could have a site with no database requirements; high performance on even free hosting; and better still, that was easy to maintain?

Well, if you’re reading this article (which, ummm, you are…) then take this as proof that all of the above is more than possible. I have just created this site using Hugo, a static site generator that takes articles written in standard markdown format, and turns them into static html.

So Why Hugo?

I have been using Hugo on a number of projects recently and I’ve been really impressed by it’s speed, simplicity and flexibility.

Given that the Hugo generates static html sites, hosting is cheap and guaranteed to be fast, even on budget hosting plans.

Give it a Try

Installing Hugo and creating your first site is much easier that even their great documentation would have you believe. If you are using a mac with homebrew and git installed, you could be up and running in well under 10 minutes.

The basic steps go something like this:

  1. Open up the Terminal and install hugo with homebrew:

    brew install hugo

  2. Create a folder for your hugo sites to live in, and cd into it:

    mkdir hugo && cd hugo

  3. Create a new site:

    hugo new site mynewsite

  4. Pick a theme from the Hugo Theme Showcase

  5. cd to the themes directory in your new site:

    cd mynewsite/themes

  6. Clone the theme repository into your themes directory (let’s use Hyde):

    git clone https://github.com/spf13/hyde.git

  7. Follow the themes installation instructions. Hint: Some themes have example sites, which you can copy to your root folder to test it out.

  8. Navigate back to your project folder:

    cd ..

  9. Start the hugo server (the -w watches for new files):

    hugo server -w

  10. As if by magic, your new (and empty) site is now available here

Now that you’re up and running, go and make some content for your new new blog. I know I need to do the same.

If you get stuck, take a look at the quickstart guide
